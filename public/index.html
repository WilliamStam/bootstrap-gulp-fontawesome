Folder structure

All my stuff lives in the "app" folder. i then point my web server to a "public" folder. but my stuff is server side. for static sites you will probably look to have a dist folder.

even my "static" files go through the backend hence why i output them in "app".

the basics of this are "your working files in 1 folder", they get processed, and dumped into another folder

your working files in "assets" (probably should be "source" tho) and the output in static (should probably go into public)

styles.scss

i like keeping my files small, easier to work on. so i devized this folder name as a file name approach. makes it easier when i only want to edit the toolbar stuff not to go through a 1000 line file

inside assets/css/styles.scss is a file variables.scss. look in the node_modules/bootstrap/scss/_variables.scss file for all the bs variables. add them to the asssets/css/variables file to override the bootstrap ones.

libraries.js

i do something similar to the styles for the js files. only there isnt an "import" facility in js so i use a list (array) of file names for it to concat and it outputs a single file with all of these

how do i use this?

run "npm install"" from the root folder (where the package.json file lives)
this will create a node_modules folder with a bunch of crap in it

gulp should now be installed. test it by running "gulp build"

you should now get an app/static folder. refresh this page. should get bootstrap running with some js

bootstrap will now be in static/css/styles.css and static/js/libraries.js so when deploying to the server dont include the nod_modules Folder

if you want it to auto run "gulp build" for you while you are working on the css / js files start off by typing "gulp watch". now whenever you edit a file in assets it will com,pile for you automaticaly. if you need to force it for whatever reason do gulp build

i included font aweosme in this. https://fontawesome.com/icons?d=gallery&s=brands,light,regular,solid&m=free

gulp build will also copy the fonts to your static folder if it goes right.. you should see an icon here

edit: app\assets\css\styles.scss\font-awesome\font-awesome.scss if you need to change the path

please note, this IS a basturdized quick sneak at what you can do with it. you will have to have different folder structuresd to this. edit the gulp file accordingly
